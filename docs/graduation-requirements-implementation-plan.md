# 졸업요건 페이지 구현 계획

## 1. 문서 목적

이 문서는 `교육·학사 > 졸업요건` 페이지를 구현하기 위한 단계별 작업 계획이다.
초기 계획 작성 뒤 구현 단계까지 진행했으며, 아래 실행 현황으로 완료 범위를 함께 관리한다.

대상 경로:

- `/ko/academics/requirements`
- `/en/academics/requirements`

핵심 목표:

- 학부와 대학원 졸업요건을 한 페이지 안에서 명확히 분리한다.
- 학부는 입학연도와 전공 유형별 학점 비교를 중심으로 구성한다.
- 대학원은 학위 취득 절차를 순서대로 이해할 수 있게 구성한다.
- 제공되지 않은 요건을 추측하거나 졸업 가능 여부를 자동 판정하지 않는다.

## 3. 예상 파일 구성

구현 전에 현재 라우팅과 컴포넌트 구조를 확인하고, 기존 파일을 우선 활용한다.

예상 데이터 파일:

- `app/data/undergraduateGraduationRequirements.ts`
- `app/data/graduateGraduationRequirements.ts`

예상 UI 변경 파일:

- `app/components/DepartmentSite.tsx`
- `app/globals.css`

컴포넌트가 지나치게 커질 경우에만 아래처럼 졸업요건 전용 파일로 분리한다.

- `app/components/requirements/GraduationRequirementsPage.tsx`
- `app/components/requirements/UndergraduateRequirements.tsx`
- `app/components/requirements/GraduateRequirements.tsx`

새 경로를 만들지 않고 기존 `academics/requirements` 분기와 연결한다.

## 4. 데이터 설계

### 4.1 학부 데이터

다음 구조를 기준으로 관리한다.

```ts
type VerificationStatus = "verified" | "needs-review";

type UndergraduateRequirement = {
  admissionYear: "2025" | "2024" | "previous";
  labelKo: string;
  labelEn: string;
  verificationStatus: VerificationStatus;
  reviewNote: string | null;
  creditSummary: MajorTypeSummary[];
  curriculumSections: CurriculumSection[];
  notices: NoticeGroup[];
  additionalPrograms: AdditionalProgramRequirement[];
};
```

`creditSummary`에는 단일전공, 학사편입, 졸업예정자 복수전공의 다음 값을 구조화한다.

- 교양기초
- 대학교양 선택
- 대학교양 필수
- 기초교육
- 교양 소계
- 전공기초
- 전공필수
- 전공선택
- 전공 소계
- 총 이수학점
- 3000·4000단위 요건

`curriculumSections`에는 2025학번 교과목 구성을 아래 순서로 저장한다.

1. 교양기초
2. 대학교양 선택
3. 대학교양 필수
4. 기초교육
5. 전공필수
6. 전공선택
7. 일반선택
8. 총 이수학점

교과목은 기존 `undergraduateCourseOfferings.ts`와 `undergraduateCourseDetails.ts`에서 코드와 명칭을 재사용한다. 졸업요건에만 존재하거나 명칭이 다른 항목은 임의로 합치지 않고 별도 검수 메모를 남긴다.

### 4.2 2024학번 처리

2025학번 데이터를 복제하지 않는다.

```ts
{
  admissionYear: "2024",
  verificationStatus: "needs-review",
  reviewNote: "2024학번 별도 졸업이수요건표 공식 확인 필요",
  creditSummary: [],
  curriculumSections: []
}
```

화면에는 다음 문구만 표시하고, 데이터 교체가 쉬운 동일 섹션 구조를 유지한다.

> 2024학번 세부 졸업이수요건은 공식 자료 확인이 필요합니다.

### 4.3 대학원 데이터

대학원 데이터는 화면 순서와 동일한 단계 배열로 관리한다.

```ts
type GraduateRequirementSection = {
  sectionId: string;
  step: string;
  titleKo: string;
  titleEn: string;
  summary: string | null;
  requirements: RequirementGroup[];
  tables: RequirementTable[];
  notes: string[];
  applicableFrom: string | null;
  reviewNote: string | null;
};
```

단계 순서:

1. 지도교수 배정
2. 수업 및 이수학점
3. 필수 교과목
4. 종합시험
5. 학위논문 예비심사
6. 학위논문 본심사
7. 영어시험
8. 학술활동
9. 기타

## 5. 단계별 구현 계획

### 단계 1. 기존 구조 확인

- [ ] `/ko/academics/requirements`와 `/en/academics/requirements`의 현재 렌더링 분기를 확인한다.
- [ ] 기존 `PageHeader`, breadcrumb, container, 왼쪽 진행 표시를 재사용할 수 있는지 확인한다.
- [ ] 졸업요건 페이지가 placeholder인지 기존 콘텐츠가 있는지 확인한다.
- [ ] 다른 페이지에 영향을 줄 수 있는 공통 CSS 선택자를 식별한다.
- [ ] 작업 전 `git status`를 확인하고 사용자 변경을 보존한다.

완료 기준:

- 기존 경로를 중복 생성하지 않고 수정 지점이 확정되어야 한다.

### 단계 2. 데이터 파일 작성

- [ ] 학부 졸업요건 데이터 타입과 데이터를 작성한다.
- [ ] 대학원 단계별 데이터 타입과 데이터를 작성한다.
- [ ] 2025학번 학점표의 숫자를 원문과 항목별로 대조한다.
- [ ] 2024학번은 `needs-review` 상태만 제공한다.
- [ ] 복수전공·부전공 기준을 별도 배열로 관리한다.
- [ ] 대학원 최소 졸업학점, 필수 교과목, 영어시험 표를 구조화한다.
- [ ] 모든 확인 필요 문구를 `reviewNote`에 기록한다.

완료 기준:

- 긴 졸업요건 원문이 페이지 컴포넌트에 하드코딩되지 않아야 한다.
- 숫자와 적용 시점을 데이터만 교체해 수정할 수 있어야 한다.

### 단계 3. 최상위 탭과 URL 상태 구현

- [ ] 학부·대학원 탭을 `role="tablist"` 구조로 구현한다.
- [ ] `?program=undergraduate`와 `?program=graduate`를 상태와 동기화한다.
- [ ] query가 없거나 잘못된 경우 학부 탭을 기본값으로 사용한다.
- [ ] 새로고침과 브라우저 이전·다음 이동 시 탭 상태를 복원한다.
- [ ] `aria-selected`, `aria-controls`, `aria-labelledby`를 연결한다.
- [ ] 좌우 방향키 또는 명확한 Tab 이동으로 탭을 선택할 수 있게 한다.

완료 기준:

- URL을 직접 열어도 올바른 과정 탭이 표시되어야 한다.

### 단계 4. 페이지 상단 구성

- [ ] 소제목 `GRADUATION REQUIREMENTS`를 표시한다.
- [ ] 제목과 소개 문구를 locale에 맞춰 표시한다.
- [ ] 옅은 블루 배경과 얇은 왼쪽 선을 사용한 안내문을 추가한다.
- [ ] 안내문이 경고 배너처럼 과하게 보이지 않게 한다.

완료 기준:

- 첫 화면에서 페이지 목적과 공식 자료 재확인 필요성을 이해할 수 있어야 한다.

### 단계 5. 학부 입학연도 선택 구현

- [ ] `2025학번`, `2024학번`, `이전 학번 안내` 선택 영역을 구현한다.
- [ ] 선택 상태를 명확한 텍스트와 선으로 표현한다.
- [ ] 2025학번은 전체 요건을 표시한다.
- [ ] 2024학번은 공식 자료 확인 필요 상태를 표시한다.
- [ ] 이전 학번은 제공된 주요 유의사항을 중심으로 안내하고 미제공 학점표를 생성하지 않는다.

권장 query 구조:

- `?program=undergraduate&year=2025`
- `?program=undergraduate&year=2024`
- `?program=undergraduate&year=previous`

완료 기준:

- 학번을 바꿔도 페이지 구조가 이동하거나 흔들리지 않아야 한다.

### 단계 6. 학부 핵심 학점 요약 구현

- [ ] 데스크톱에서는 단일전공·학사편입·졸업예정자 복수전공 비교표를 제공한다.
- [ ] 총 이수학점과 전공 소계를 가장 강하게 강조한다.
- [ ] 전공필수·전공선택·상위단위 요건을 빠르게 비교할 수 있게 한다.
- [ ] 모바일에서는 유형별 세로 요약으로 변환한다.
- [ ] 가로 스크롤 없이 모든 항목을 읽을 수 있게 한다.

완료 기준:

- 학생이 자신의 유형별 총학점과 전공학점을 첫 번째 정보 블록에서 찾을 수 있어야 한다.

### 단계 7. 2025학번 교과목 구성 구현

- [ ] 교양기초, 대학교양, 기초교육, 전공필수, 전공선택을 구분한다.
- [ ] 각 영역 제목에 필수 또는 최소 학점을 함께 표시한다.
- [ ] 전공선택 31개 항목은 과도한 카드 없이 읽기 좋은 목록으로 구성한다.
- [ ] 기존 교과목 데이터와 매칭되는 항목은 해당 데이터를 재사용한다.
- [ ] 가능한 경우 교과목 안내 페이지의 검색 query로 연결한다.
- [ ] 코드 불일치나 명칭 변경 가능성이 있는 항목은 자동 정정하지 않는다.

권장 교과목 링크:

```text
/ko/academics/courses?tab=schedule&q=MEU2600
```

완료 기준:

- 교과목명이 중복 데이터로 흩어지지 않고 기존 교과목 정보와 연결되어야 한다.

### 단계 8. 학부 유의사항과 복수전공·부전공 구현

- [ ] 주요 유의사항을 세 주제로 분리한다.

- [ ] 강한 경고색을 반복하지 않고 제목, 얇은 선, 여백으로 구분한다.
- [ ] 복수전공·부전공 학점을 비교표로 표시한다.
- [ ] 필수 이수과목 기준은 입학연도별로 구분한다.

완료 기준:

- 긴 유의사항을 모두 펼치지 않아도 주제와 적용 대상을 파악할 수 있어야 한다.

### 단계 9. 대학원 단계형 UI 구현

- [ ] 모바일에서는 9개 단계를 세로 아코디언으로 변환한다.
- [ ] 최소 졸업학점은 과정별 비교표로 표시한다.
- [ ] 대학원 필수 교과목은 코드, 과목명, 학점, 적용 시점을 구분한다.
- [ ] 종합시험과 논문심사는 자격, 시기, 심사위원, 합격기준을 소제목으로 나눈다.
- [ ] 영어시험은 시험 종류별 점수 비교표로 표시한다.
- [ ] 학술활동과 기타 항목은 확인 경로를 포함한 간결한 안내로 구성한다.

완료 기준:

- 대학원생이 현재 단계의 선행조건과 다음 단계를 순서대로 이해할 수 있어야 한다.

### 단계 10. 반응형과 접근성 보완

- [ ] 데스크톱 비교표와 단계 내비게이션을 점검한다.
- [ ] 태블릿에서 표 열과 단계 목록이 지나치게 좁아지지 않게 한다.
- [ ] 모바일에서는 비교표를 행 또는 유형별 세로 구조로 변환한다.
- [ ] 긴 과목명과 영문명이 화면 밖으로 넘치지 않게 한다.
- [ ] 모든 탭, 링크, 아코디언에 `focus-visible`을 제공한다.
- [ ] 표에는 `caption`, `thead`, `tbody`, `scope`를 사용한다.
- [ ] 아코디언 상태와 패널 관계를 스크린리더에 전달한다.
- [ ] 색상만으로 선택 상태를 표현하지 않는다.

완료 기준:

## 7. locale 처리 원칙

- 한국어 페이지는 제공된 공식 원문을 사용한다.
- 공통 UI 라벨과 짧은 안내는 한국어·영어를 제공한다.
- 공식 영문 규정이 없는 긴 졸업요건은 임의 번역하지 않는다.
- 영어 페이지에는 한국어 원문과 번역 준비 중 안내를 표시할 수 있게 데이터 구조를 설계한다.
- 추후 공식 번역문이 제공되면 데이터만 교체할 수 있게 한다.

## 8. 완료 정의

다음 조건을 모두 만족하면 구현 완료로 판단한다.

- 학부와 대학원 탭이 query와 동기화된다.
- 2025학번 데이터가 제공 원문과 일치한다.
- 2024학번 데이터가 2025학번으로 복제되지 않는다.
- 학부 핵심 학점, 교과목 구성, 유의사항, 복수전공·부전공을 확인할 수 있다.
- 대학원 9단계와 관련 표를 확인할 수 있다.
- 모바일에서 가로 스크롤 없이 읽을 수 있다.
- 키보드와 스크린리더 구조를 지원한다.
- 수정 금지 영역에 변경이 없다.
- 빌드가 성공한다.
